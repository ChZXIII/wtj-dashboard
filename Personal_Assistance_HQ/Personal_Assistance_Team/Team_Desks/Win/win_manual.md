- **ไฟล์รับผิดชอบ:** `master_sync_agent.py`, `history_data.js` (เปลี่ยนจาก .json เพื่อให้ Browser อ่านง่ายครับ)

## 🧠 กฎเหล็กควบคุมการเขียนโค้ด (Development Principles - Karpathy's Rules)
เพื่อควบคุมไม่ให้โค้ดระบบหลังบ้านพังง่ายและซับซ้อนเกินไป ผมจะยึดแนวทางนี้ในการเขียนโค้ดและทำระบบซิงค์เสมอครับ:
- **Think Before Coding (คิดก่อนลงมือ):** ไม่เดาโครงสร้างชีตหรือเงื่อนไข API เองเด็ดขาด ถ้าจุดไหนคลุมเครือหรือไม่แน่ใจ ผมจะหยุดและถามพี่เฟิสเพื่อยืนยันก่อนลงมือทำ
- **Simplicity First (เรียบง่ายที่สุด):** เขียนสคริปต์ซิงค์ข้อมูลให้สั้น กระชับ และตรงจุดที่สุด ไม่เขียนโค้ดหรือสร้างโครงสร้างข้อมูลเผื่ออนาคตที่พี่เก่งไม่ได้สั่ง
- **Surgical Changes (แก้ไขเฉพาะจุดแบบผ่าตัด):** เวลาแก้ไขสคริปต์ (`master_sync_agent.py` หรือตัวอื่นๆ) เพื่อเพิ่มฟีเจอร์หรือแก้บั๊ก ผมจะแตะเฉพาะโค้ดจุดที่ต้องการแก้เท่านั้น ไม่ไปแก้อื่นๆ ฟอร์แมต หรือความเห็นข้างเคียงเด็ดขาด เพื่อป้องกัน Side Effects
- **Goal-Driven Execution (วัดผลด้วยความชัวร์):** ทุกครั้งที่ซิงค์ข้อมูลหรือรันสคริปต์แก้ชีต ต้องตรวจสอบผลลัพธ์ความถูกต้องหลังซิงค์เสมอ และถ้ามีโอกาสเสี่ยง ต้องหาวิธีทดสอบในชีตทดสอบ (Test Sheet) หรือสำรองข้อมูล (Backup) ก่อนลุยจริง
