# 🎬 WTJ Story — Official Content Production Workflow (ฉบับปรับปรุงใหม่)

## 📌 หลักการสำคัญ
- **แกคุยกับเฟิสเท่านั้น** เฟิสจะประสานงานกับทีมทุกคนและดันงานผ่านท่อส่งงาน (Pipeline) ให้เอง
- **ทุกขั้นตอนต้องผ่านการตรวจสอบ** ก่อนส่งต่อขั้นถัดไปเสมอเพื่อรับประกันคุณภาพและ Fact ความถูกต้อง
- **พี่คริส (ผู้กำกับ) และน้องพี (คนทำภาพ):** เนื่องจากช่วงนี้ยังไม่มีงานภาพประกอบ ให้คู่มือและเอเจนต์ทั้งสองท่านนี้สแตนด์บาย (On-hold) ไว้ก่อน เมื่อเริ่มลุยงานภาพภาพยนตร์เมื่อไหร่ ค่อยเพิ่มขั้นตอนของทั้งคู่กลับมาทำงานร่วมกันอีกทีครับ

## 🔄 Pipeline ทั้งหมด (2 รูปแบบหลัก)

ทีมคอนเทนต์ของเราทำงานครอบคลุม 2 โปรเจกต์หลัก ซึ่งแต่ละโปรเจกต์จะใช้เส้นทางทำงานดังนี้:

### รูปแบบที่ 1: WTJ Story (สคริปต์วิดีโอเจาะลึกอาชีพ/เรื่องราว)
> **Pipeline:** แก -> เดียร์ (Idea Card) -> น้ำ (Creative Angle) -> มิวสิค (Marketing/Meme) -> ครีม (Research) -> ออ (Audit) -> เรย์ (Writer) <-> มิวสิค (Content Optimize) -> คริ (Critique Script) -> เฟิส (จัดเก็บ)

1. **เดียร์ (Idea Architect):** รับหัวข้อจากแกมาแปลงเป็น Idea Card (หาแก่นกับประเด็นพื้นฐาน)
2. **น้ำ (Creative & Content Creator):** เอาไอเดียการ์ดมาคิด "มุมมองการนำเสนอและรูปแบบการเล่าเรื่องที่แปลกใหม่ว้าวซ่า" (Creative Concept)
3. **มิวสิค (Marketer - รอบแรก):** สแกนไอเดียของน้ำแล้วเติมแนวทางมีม/กระแสกระแสโลกออนไลน์ (Real-time Marketing) รวมถึงวางแผนแคมเปญสปอนเซอร์เบื้องต้น
4. **ครีม (Researcher):** ค้นคว้าวิจัยข้อมูลเชิงลึกและแหล่งอ้างอิงอิงตามแนวทางของน้ำและมิวสิค
5. **ออ (Auditor):** ตรวจสอบความถูกต้องของ Fact ข้อมูลวิจัยของครีม
6. **เรย์ (Writer):** เอาข้อมูลวิจัยที่ตกผลึกแล้วมาแต่งโครงร่างสคริปต์วิดีโอ YouTube และร่างโพสต์ Facebook
7. **มิวสิค (Marketer - รอบเขียนบท):** ทำงานร่วมมือกับเรย์เพื่อขยี้ Hook (เปิดตัวคลิป), ตบพาดหัวตัวหนังสือ, และปรับแต่งมุกมีมในบทให้เด่นสะดุดตา
8. **คริ (Critic - รอบสคริปต์):** วิจารณ์และตรวจทานท่อนไหลและคุณภาพของสคริปต์ร่างสุดท้าย
9. **เฟิส (Coordinator):** รวบรวมไฟล์ทั้งหมดส่งเข้า Apple Notes และอัปเดตระบบความรู้ของทีม


---

### รูปแบบที่ 2: WTJ Podcast / Talk (รายการสนทนาเชิงลึก)
> **Pipeline:** แก -> เดียร์ -> น้ำ -> มิวสิค -> ครีม -> ออ -> เรย์ <-> มิวสิค -> คริ -> เฟิส

1. **เดียร์:** วางโครงสร้างประเด็นสัมภาษณ์พื้นฐาน
2. **น้ำ (ครีเอทีฟ):** ดีไซน์รูปแบบการจี้ถามหรือมุมกล้องการนำเสนอคำถามให้น่าสนใจและสะดุดใจ
3. **มิวสิค:** วางแผนกิจกรรมแคมเปญโปรโมต หรือวางแพ็กเกจสปอนเซอร์โฆษณาที่เหมาะกับแขกรับเชิญ
4. **ครีม:** รีเสิร์ชข้อมูลเชิงลึกของตัวแขกรับเชิญและอาชีพของเขา
5. **ออ:** ตรวจสอบความถูกต้องของ Fact เพื่อไม่ให้สัมภาษณ์ผิดประเด็น
6. **เรย์ (นักเขียน):** เรียบเรียงชุดคำถามให้ลื่นไหลเป็นธรรมชาติเหมือนบทสนทนาจริง (Interview Flow)
7. **มิวสิค (รอบเขียนบท):** ร่วมขยี้ท่อนเปิดสัมภาษณ์ให้น่าดึงดูด และตรวจพาดหัวโปรโมตคลิปสั้นไฮไลต์
8. **คริ (รอบสคริปต์):** วิจารณ์ตรวจสอบ Flow คำถามในบทสัมภาษณ์รอบสุดท้าย
9. **เฟิส:** บันทึกและส่งออกข้อมูล

## 📊 เวิร์กโฟลว์ข้อมูลวิเคราะห์ป้อนกลับ (Data Feedback Loop)
เพื่อป้องกันไม่ให้ทีมผลิตงานแบบคาดเดาไปเอง น้องปาย (Pie - Data Analyst) จะทำการวิเคราะห์ข้อมูลหลังบ้านแล้วป้อนกลับเข้าสู่กระบวนการดังนี้:

1. **ดึงข้อมูลสถิติ:** น้องปายใช้สคริปต์ `skills/performance_analyzer.py` สกัดตัวเลขสถิติจาก YouTube/Facebook รายสัปดาห์
2. **สร้างรายงานวิเคราะห์:** บันทึกรายงาน `data_analysis_report.md` ใน Workspace เพื่อระบุพฤติกรรมผู้ชม จุดที่น่าปรับแก้ (เช่น นาทีที่คนกดปิดคลิปเยอะ) และกระแสโพสต์ที่ปัง
3. **ส่งต่อแบรนดิ้ง/ครีเอทีฟ:** น้องปายสรุปจุดบกพร่อง/โอกาสทองส่งต่อให้ **น้องเดียร์, น้องน้ำ และน้องมิวสิค** นำไปปรับไอเดีย หัวข้อ และกลยุทธ์การเขียนบทของตอนต่อไป
4. **ส่งต่อแดชบอร์ด:** ส่งค่าตัวเลขสรุปยอดรายสัปดาห์ให้น้องเอ็ม (M) ทำการแสดงผลอัปเดตหน้าจอควบคุม (WTJ Dashboard)
