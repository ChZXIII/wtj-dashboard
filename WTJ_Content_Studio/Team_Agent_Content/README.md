# Team Agent Content Workspace

นี่คือพื้นที่ทำงานสำหรับทีมงาน AI ที่ช่วยผลิตคอนเทนต์ให้กับโปรเจกต์ "What the job" โดยแยกสัดส่วนชัดเจนจากการทำงานของเลขา (First)

## สมาชิกในทีม
1. **คริส (Chris)** - ผู้กำกับ (Director) [ผู้หญิง]: คอยคุมทิศทางของเนื้อหา เน้นความเป็นธรรมชาติ ไม่เป็นทางการ และอิงข้อมูลเชิงลึก
2. **ครีม (Cream)** - หัวหน้านักวิจัย (Lead Researcher) [ผู้หญิง]: ค้นหาข้อมูลเชิงลึก (Insight) ข้อเท็จจริง ตัวเลขรายได้ และตรวจสอบแหล่งอ้างอิง
3. **น้ำ (Nam)** - ครีเอทีฟและคอนเทนต์ครีเอเตอร์ (Creative & Content Creator) [ผู้หญิง]: สาวมั่น ไอเดียบรรเจิด เชี่ยวชาญ TikTok และการทำคอนเทนต์ให้ Viral
4. **ปาย (Pie)** - นักวิเคราะห์ข้อมูลหลังบ้าน (Data Analyst) [ผู้หญิง]: วิเคราะห์สถิติจุดคนปิดคลิป (Retention Curve), CTR, และการเข้าถึงเพื่อชี้เป้าโอกาสใหม่ๆ

## โครงสร้างโฟลเดอร์
- `director_agent.py`: สคริปต์หลักสำหรับเรียกใช้งานและพูดคุยกับผู้กำกับผ่าน Terminal
- `pie_agent.py`: สคริปต์หลักสำหรับคุยและขอคำวิเคราะห์ตัวเลขกับน้องปายผ่าน Terminal
- `prompts/`: เก็บไฟล์บรีฟ (System Prompt) ของ Agent แต่ละตัว เพื่อให้เข้าไปแก้ไขได้ง่ายๆ (เช่น `director_prompt.md`)
- `workspace/`: 
  - `1_raw_materials/`: โฟลเดอร์สำหรับใส่ข้อมูลดิบจากการสัมภาษณ์ หรือรีเสิร์ช
  - `2_drafts/`: สคริปต์ร่างที่ยังรอการปรับแก้
  - `3_final_scripts/`: สคริปต์สมบูรณ์พร้อมนำไปพากย์
  - `data_analysis_report.md`: รายงานวิเคราะห์สถิติล่าสุดโดยน้องปาย
- `skills/`: คลังท่าไม้ตาย (Skill Library) เก็บสคริปต์อัตโนมัติสำหรับเพิ่มความสามารถให้เอเจนต์
  - `youtube_researcher.py`: ค้นหาเทรนด์และอินไซต์จาก YouTube (สำหรับครีม)
  - `youtube_transcriber.py`: ถอดเทปเสียงและเนื้อหาจากลิงก์ YouTube เป็นข้อความภาษาไทย
  - `source_vault_compiler.py`: สรุปและเชื่อมโยงคลังข้อมูลดิบเข้าเป็นแหล่งความรู้อย่างเป็นระเบียบ
  - `social_media_extractor.py`: สกัดและยกร่างโพสต์โซเชียลมีเดียจากสคริปต์หรือเทปถอดความ
  - `setup_notes_folders.py`: ตรวจสอบและสร้างโครงสร้างโฟลเดอร์สำหรับตรวจงานบน Apple Notes (`WTJ -> Facebook -> [1_Drafts, 2_Approved, 3_Published]`) อัตโนมัติ
  - `fb_draft_generator.py`: สแกนหาทรานสคริปต์ล่าสุดใน Apple Notes (โน้ตที่มีคำว่า "Transcript:") เรียกใช้งานเอเจนต์ (น้ำ & เรย์) ปั้นโพสต์ 3 รูปแบบ (วันพุธ Teaser, วันศุกร์ Quote, วันเสาร์ Reels) แล้วยิงเข้าแอปโน้ตแต่ละวันของพี่เก่งอัตโนมัติ
  - `facebook_publisher.py`: บอทโพสต์คอนเทนต์ขึ้นเพจ Facebook ดึงโพสต์ที่อนุมัติจาก Apple Notes โฟลเดอร์ `2_Approved` ไปโพสต์เพจจริง และย้ายโน้ตไปเก็บสำรองประวัติใน `3_Published` อัตโนมัติ (รองรับระบบ Dry-Run สำหรับทดสอบ)
  - `performance_analyzer.py`: บอทวิเคราะห์สถิติวิดีโอ YouTube และเพจ Facebook (สำหรับน้องปาย) ออกรายงาน Insights และระบุความผิดปกติ
  - `podcast_notes_publisher.py`: บอทส่งเนื้อหา/บทสัมภาษณ์ หรือสคริปต์ของทีมผลิตเข้า Apple Notes แยกรายหัวข้อแบบจัดระเบียบอัตโนมัติ

## วิธีใช้งาน
1. เปิด Terminal ชี้มาที่โฟลเดอร์นี้
2. ตั้งค่า API Key: `export GOOGLE_API_KEY="your_api_key_here"` 
3. เรียกใช้ Skill (ตัวอย่าง): `python skills/performance_analyzer.py`
4. รันคำสั่งคุยกับเอเจนต์ (ตัวอย่าง): `python director_agent.py` หรือ `python pie_agent.py` เพื่อคุยวิเคราะห์สถิติกับปาย

## 🗣️ Tone of Voice & Communication Policy
- **ห้ามใช้ "ครับ":** ทีมงานทุกคนเป็นเพื่อนซี้กัน ให้ใช้ภาษาที่เป็นกันเอง (แก/ชั้น, นะจ๊ะ, จ้า) เท่านั้น เพื่อความเป็นธรรมชาติและลื่นไหลในการทำงาน
- **ความเป็นกันเอง:** เน้นการสื่อสารเหมือนคุยกับเพื่อนสนิท (Buddy Style) ไม่ต้องมีพิธีรีตองเยอะ
- **นโยบายฟีดแบ็กที่ตรงไปตรงมา (Honest Critique Policy):** ห้ามอวยกันเองอย่างเดียวเด็ดขาด! ให้พูดความจริงต่อกันอย่างตรงไปตรงมา ใครมีคอมเมนต์อะไรให้บอกทั้งข้อดี ข้อเสีย และสิ่งที่ควรปรับปรุง เพื่อให้ผลงานได้รับการพัฒนาอย่างต่อเนื่อง ให้น้องๆ ทุกคนถกเถียงคุยกันเพื่อสิ่งที่ดีที่สุด

---
**หมายเหตุเพิ่มเติมจากเลขาเฟิส:** พื้นที่นี้สงวนไว้สำหรับงานผลิตเนื้อหา YouTube เท่านั้นนะจ๊ะ ไม่เกี่ยวข้องกับงานผู้ช่วยส่วนตัว (Win/Jane) จ๊ะ!

## 🖥️ ระบบ Dashboard (Veda Tactical Terminal)
โฟลเดอร์ `dashboard/` คือกระดานควบคุมของทีมงาน ซึ่งถูกออกแบบมาในสไตล์ **"Celestial Being Tactical Schematic"** (Gundam 00)
- **Aesthetic Rules (กฎการออกแบบ):**
  - **Schematic Wireframe:** เน้นความคลีนแบบพิมพ์เขียว พื้นหลังตาราง (Grid) บางๆ โทนสว่าง
  - **Veda Mode (Dark Theme):** รองรับระบบสลับโหมดมืด (Dark Mode) สำหรับสถานการณ์ฉุกเฉิน
  - **Sharp Edges & Hard Shadows:** หน้าต่างข้อมูลต้องเป็นกรอบเหลี่ยม (คมกริบ ขอบโค้งมนแค่ 2px) และมีเงาทึบทะแยงมุม (Hard Drop Shadow) ลอยเด่นเหนือแผนที่
  - **Floating Window System:** ระบบ Navigation ด้านซ้าย *ไม่ใช่การเปลี่ยนหน้าเว็บ* แต่จะเป็นการเรียก **"Pop-up Window"** เด้งขึ้นมาซ้อนเหลื่อมกันตรงกลางหน้าจอ เพื่อจำลองหน้าต่างโฮโลแกรมซ้อนทับกันแบบในอนิเมะ
- **ไฟล์สำคัญ:**
  - `dashboard/index.html`: โครงสร้างหลักของ Veda Terminal
  - `dashboard/style.css`: หัวใจหลักของความคลีนแบบพิมพ์เขียว (ห้ามใส่เงาฟุ้งแบบมือถือเด็ดขาด)
  - `dashboard/app.js`: ระบบคุมหน้าต่าง Floating Windows แบบซ้อนทับกัน (Z-index Management)
