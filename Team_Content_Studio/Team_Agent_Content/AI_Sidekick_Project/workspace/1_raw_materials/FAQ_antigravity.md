## ข้อ 1 — การติดตั้ง Antigravity

**⚠️ ก่อนอื่น: URL ที่ถูกต้องคือ `antigravity.google` ไม่ใช่ antigravity.ai (อันนั้นคือโดเมนขายปกติ!)**

**ขั้นตอนติดตั้ง:**
1. เข้า → `antigravity.google`
2. กด Download (มีให้เลือก Mac / Windows / Linux)
3. ติดตั้งโปรแกรมตามปกติ (คลิกถัดไปไปเรื่อยๆ)
4. ล็อกอินด้วย Google Account
5. เลือก Workspace (โฟลเดอร์ที่จะทำงาน) แล้วเริ่มใช้ได้เลย

**ข้อดีเหนือ Claude Code:** ไม่ต้องติดตั้ง Git, Node.js, Python เพิ่มเติมเลย!

## ข้อ 2 — Product ใน Antigravity ทั้ง 4 ตัว ต่างกันยังไง?

ตั้งแต่ปี 2026 Google แยก Antigravity ออกเป็น **4 ผลิตภัณฑ์** แต่ละตัวมีจุดประสงค์ต่างกัน:

| Product | คืออะไร | เหมาะกับใคร |
|---|---|---|
| **Antigravity 2.0** | App ตั้งโต๊ะสำหรับจัดการ Agent หลายตัวพร้อมกัน ไม่มี Code Editor | คนที่ต้องการสั่งงาน AI เป็นทีม ทำงานซ้ำอัตโนมัติ |
| **Antigravity IDE** | ตัวเดิมที่เป็น VS Code-based มี Editor + Terminal ครบ | Developer / คนที่ต้องแตะไฟล์โค้ดโดยตรง |
| **Antigravity CLI** | ใช้ผ่าน Terminal พิมพ์คำสั่งตรงๆ ไม่มีหน้าต่าง UI | Developer สาย Keyboard / ระบบ Server |
| **Antigravity SDK** | Library Python ให้นักพัฒนาสร้าง Agent บน Infrastructure ตัวเอง | บริษัทที่ต้องการสร้าง AI ฝังในระบบตัวเอง |

**สำหรับคอร์สของแก และกลุ่มเป้าหมาย (ออฟฟิศ, ธุรกิจ, Freelance):**
> ใช้แค่ **Antigravity 2.0** ก็พอ! มันทำได้ทุกอย่างที่ต้องการโดยไม่ต้องแตะโค้ดเลย  
> CLI และ SDK ไว้สอนในส่วน Advanced สำหรับคนที่อยากต่อยอด

## ข้อ 3 — Settings หลังติดตั้ง ตั้งค่าอะไรได้บ้าง?

**Settings สำคัญที่ต้องรู้:**

**🤖 Model Selection** — เลือกโมเดล AI ที่ใช้
- Gemini 3.1 Pro → ฉลาดที่สุด แต่กิน Token เยอะ
- Gemini 3.5 Flash → เร็ว ประหยัด เหมาะงานทั่วไป
- Claude Sonnet → ทางเลือกจาก Anthropic
- *แนะนำสำหรับมือใหม่: Flash ก่อน แล้วค่อยสลับ Pro เมื่องานซับซ้อน*

**📁 Workspace** — โฟลเดอร์ที่ AI จะมองเห็นและทำงานด้วย  
*สำคัญมาก: AI เห็นแค่ไฟล์ในโฟลเดอร์ที่เปิดไว้เท่านั้น*

**🌐 Browser Integration** — เปิดให้ AI เปิดเว็บและดูผลลัพธ์ได้  
*เปิดไว้เลย มีประโยชน์มาก*

**🔌 MCP Connections** — เชื่อมต่อกับ Tool ภายนอก เช่น NotebookLM  
*ข้อนี้สำหรับ Advanced user — ข้ามไปก่อนได้*

## ข้อ 4 — ขั้นตอนสร้าง Agent ทีละ Step (แบบละเอียด)

> **Agent = โปรเจกต์ที่มี GEMINI.md + มี "บุคลิก" และ "หน้าที่" ชัดเจน**

**Step 1:** สร้างโฟลเดอร์ใหม่บนคอมแก  
*(ตั้งชื่อให้สื่อ เช่น `Marketing_Agent`, `Sales_Agent`)*

**Step 2:** เปิดโฟลเดอร์นั้นใน Antigravity  
*(File → Open Folder → เลือกโฟลเดอร์ที่สร้าง)*

**Step 3:** บอก AI ว่า *"ช่วยสร้างไฟล์ GEMINI.md ให้หน่อย"*  
AI จะสร้างไฟล์นี้ให้เป็น "บัตรประจำตัว" ของโปรเจกต์

**Step 4:** ใส่ข้อมูลใน GEMINI.md เช่น:
- Agent นี้ทำหน้าที่อะไร?
- พูดในโทนไหน?
- มีกฎอะไรที่ต้องรู้?
- ตัวตนของเราเป็นแบบไหน?

**Step 5:** ทดสอบ — พิมพ์ถามอะไรก็ได้ แล้วดูว่า AI ตอบตรงกับที่ตั้งค่าไว้ไหม

**Step 6:** เพิ่ม Skills และ Commands ตามต้องการ

## ข้อ 5 — Skill / SOP คืออะไร? มีเคสตัวอย่างอะไร?

**Skill = คู่มือทำงาน (SOP) ที่ให้ AI อ่านแล้วทำตามได้ทุกครั้ง**

ถ้าเทียบให้เห็นภาพ:
- คุณจ้างพนักงานใหม่มา 1 คน
- แทนที่จะอธิบายทุกครั้งว่า "ทำงานยังไง"
- คุณแค่ยื่น "คู่มือพนักงาน" ให้อ่าน → แล้วเขาทำได้ถูกต้องเลย
- **Skill ก็คือคู่มือนั้น แต่อยู่ในรูปไฟล์ข้อความ**

**ตัวอย่าง Skill จริงๆ:**

| Skill | ทำอะไร |
|---|---|
| `สรุปประชุม` | รับ transcript ประชุม → สรุป MOM พร้อม Action Items |
| `เขียนโพสต์ FB` | รับ topic → เขียน Caption + Hashtag ในสไตล์แบรนด์ |
| `วิเคราะห์คู่แข่ง` | รับชื่อแบรนด์ → รีเสิร์ชและสรุปจุดแข็ง/จุดอ่อน |
| `ตอบอีเมลลูกค้า` | รับอีเมล → ร่างคำตอบในโทนมืออาชีพ |
| `ทำ Content Plan` | รับธีม → วางแผนโพสต์ 30 วัน |

**วิธีสร้าง Skill:**
บอก AI ว่า *"สร้าง Skill สำหรับ [งานที่อยากทำ] ให้หน่อย"*  
AI จะสร้างไฟล์ใน folder `skills/` ให้อัตโนมัติ

## ข้อ 6 — Skill กับ SOP ต่างกันยังไง?

**ตอบตรงๆ: ใน Antigravity มันคือสิ่งเดียวกัน**

| | ในโลกธุรกิจ | ใน Antigravity |
|---|---|---|
| **SOP** | Standard Operating Procedure (เอกสารกระดาษ) | เนื้อหาข้างในไฟล์ Skill |
| **Skill** | ไม่มีคำนี้ | ไฟล์ที่เก็บ SOP เอาไว้ |

**พูดง่ายๆ:** Skill = ไฟล์ที่บรรจุ SOP เอาไว้  
เราแค่เรียกว่า "Skill" เพราะมันทำให้ AI มีความสามารถ (Skill) ใหม่เพิ่มขึ้น

## ข้อ 7 — /Commands ต้องใช้ไหม? ปุ่มลัดไปดูได้ที่ไหน?

**ตอบ: ไม่บังคับ แต่มีประโยชน์มากถ้าทำงานซ้ำบ่อยๆ**

**Slash Commands ที่ติดมาให้แล้ว:**

| Command | ใช้ทำอะไร |
|---|---|
| `/goal` | สั่งงานยาวๆ แบบ "ทำให้เสร็จ อย่าหยุด" |
| `/schedule` | ตั้งเวลาให้ AI ทำงานอัตโนมัติ |
| `/browser` | สั่งให้ AI ใช้เว็บเบราว์เซอร์ |
| `/grill-me` | ให้ AI ถามเพื่อช่วยวางแผนงาน |

**วิธีดูทั้งหมด:** พิมพ์ `/` ในช่องแชท แล้ว Antigravity จะแสดง list ขึ้นมาเลย

**สร้าง Custom Command เพิ่มเองได้:** บอก AI ว่า *"ช่วยสร้าง Slash Command ชื่อ /[ชื่อ] ที่ทำ [อะไร]"*

## ข้อ 8 — ทำ Subagent ยังไง?

**Subagent = ส่ง "ลูกน้อง AI" ออกไปทำงานพร้อมกันหลายตัว**

**วิธีใช้งาน (ไม่ต้องเขียนโค้ด):**  
แค่บอก AI ว่างานมีหลายส่วน แล้วบอกให้แบ่งไปทำพร้อมกัน

ตัวอย่าง Prompt:  
> *"รีเสิร์ชคู่แข่งให้ 3 เจ้าพร้อมกันเลย แล้วสรุปมาให้ทีละเจ้า"*

AI จะส่ง Subagent ออกไป 3 ตัว ทำงานพร้อมกัน แทนที่จะทำทีละอัน

**ดูสถานะ Subagent:** จะมีแถบแสดงสถานะให้เห็นว่า Agent ไหนกำลังทำอะไรอยู่

**⚠️ ข้อควรระวัง:** Subagent กิน Token เยอะกว่าปกติ ใช้เฉพาะเมื่องานต้องทำแบบขนานจริงๆ

## ข้อ 9 — Context เต็มแล้วจะเกิดอะไรขึ้น?

**Context = ความจำของ AI ในการสนทนาหนึ่งๆ**  
เหมือน "กระดาษทด" ที่ AI ใช้จำสิ่งที่คุยกันมา — มันมีขนาดจำกัด

**อาการที่เริ่มเกิดขึ้นเมื่อ Context ใกล้เต็ม:**
- AI เริ่มลืมสิ่งที่คุยไปก่อนหน้า
- คำตอบเริ่มเพี้ยน หรือขัดแย้งกับที่เคยตอบ
- ประสิทธิภาพลดลงชัดเจน

**วิธีแก้:**
1. **เริ่ม Conversation ใหม่** — ไฟล์งานทั้งหมดยังอยู่ แค่ประวัติแชทหายไป
2. **อ้างอิง File แทนการวาง Text** — บอก AI ให้ไปอ่านไฟล์แทนการวางข้อความยาวๆ ในแชท
3. **แตกงานเป็นชิ้นเล็กๆ** — ทำทีละส่วน อย่าสั่งงานใหญ่ครั้งเดียว

## ข้อ 10 — ข้อจำกัดสำคัญของ Antigravity

| ข้อจำกัด | รายละเอียด | วิธีรับมือ |
|---|---|---|
| **Context Window** | AI มีความจำจำกัดในการสนทนา | เริ่ม Conversation ใหม่บ่อยๆ |
| **Rate Limit** | แพลนฟรีมีขีดจำกัดต่อชั่วโมง | ใช้งานจริงๆ เปลี่ยน Pro |
| **Hallucination** | AI อาจแต่งข้อมูลที่ไม่มีจริง | ตรวจสอบข้อมูลสำคัญเสมอ |
| **ไม่มี Real-time Data** | AI ไม่รู้ข่าวสดหรือราคาหุ้นปัจจุบัน | ป้อนข้อมูลสดให้ AI เอง |
| **ไม่ใช่ผู้เชี่ยวชาญเฉพาะทาง** | กฎหมาย / การแพทย์ ใช้ตรวจสอบก่อนเสมอ | ใช้เป็น Draft แล้วให้ผู้เชี่ยวชาญตรวจ |

## ข้อ 11 — เชื่อมกับ Gemini Products อื่นได้ไหม?

**ตอบ: เชื่อมได้ผ่าน MCP (Model Context Protocol)**

Google ecosystem ที่เชื่อมได้:
- ✅ **NotebookLM** — ผ่าน MCP Server
- ✅ **Google Drive** — อ่าน/เขียนไฟล์
- ✅ **Google Docs/Sheets** — ดึงข้อมูล
- ✅ **Google Calendar** — ดูกำหนดการ
- ⚠️ **Gemini App** — ใช้แยกกัน ยังไม่ได้ integrate ตรง

**แต่สำหรับมือใหม่:** ไม่ต้องรีบ! ใช้ Antigravity เดี่ยวๆ ก่อน ทำได้เยอะมากแล้ว  
MCP เป็นเรื่อง Advanced ที่จะสอนใน Lesson ถัดไป

## ข้อ 12 — เชื่อม NotebookLM ได้ไหม?

**ตอบ: ได้! แต่ต้องตั้งค่า MCP Server ก่อน**

**Workflow ที่ดีที่สุด (ไม่ต้องเชื่อมก็ได้):**
```
NotebookLM → ถอดเทป / สรุปข้อมูล → Copy ผลลัพธ์ → วางใน Antigravity
```

เหมือนที่แกทำกับ VDO ที่ส่งมาให้ชั้นดูนั่นแหละ! มันคือ Workflow ที่ใช้ได้ผลจริง

**ถ้าอยากให้ทำอัตโนมัติ:** ตั้ง MCP Connection  
*(ยากขึ้น แต่สอนได้ใน Advanced Lesson)*

## ข้อ 13 — Antigravity เหมาะกับใครบ้าง?

**✅ เหมาะมากสำหรับ:**
- พนักงานออฟฟิศที่ทำงานซ้ำๆ ต้องการผู้ช่วย
- เจ้าของธุรกิจที่อยากประหยัดเวลา ลดต้นทุนคน
- Freelance ที่ต้องการส่งงานเร็วขึ้น คุณภาพดีขึ้น
- คนที่ต้องเขียนเยอะ (Content, อีเมล, รายงาน)
- คนที่ทำงานกับข้อมูล/เอกสารเยอะๆ

**⚠️ อาจไม่เหมาะกับ:**
- คนที่ต้องการ Real-time data แบบ 100% (หุ้น, ข่าว)
- คนที่ต้องการความลับสูงมาก (AI อ่านไฟล์ที่เราป้อนให้)
- คนที่คาดหวังว่า AI จะไม่มีวันผิด (ต้องตรวจสอบเสมอ)
