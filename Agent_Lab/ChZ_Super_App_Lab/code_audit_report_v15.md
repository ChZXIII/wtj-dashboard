## 🟡 ข้อเสนอแนะ (Advisory)

- **1. ข้อแนะนำเรื่องการป้องกันข้อมูลซ้ำซ้อนใน LocalStorage เมื่อบันทึกลูกค้าที่ไม่มีเลขประจำตัวผู้เสียภาษี (Empty TaxID Duplicate Prevention):**
  - **พิกัดโค้ด:** [super_app_anime.html:L6831-6832](file:///Users/chz/Desktop/ChZ_Agent_Corp/Agent_Lab/super_app_anime.html#L6831-L6832)
  - **การแกะรอยโค้ด (Path Tracing):**
    ในขั้นตอนการเปรียบเทียบข้อมูลก่อนบันทึก:
    ```javascript
    const existingIdx = customClients.findIndex(c => c.taxId === taxId && taxId !== '');
    const existsInDefaults = DEFAULT_CLIENTS.some(c => c.taxId === taxId && taxId !== '');
    ```
    หากผู้ใช้ต้องการแก้ไขหรืออัปเดตข้อมูลลูกค้าเดิมที่ **ไม่มีเลขประจำตัวผู้เสียภาษี (Tax ID ว่าง)** เมื่อกดบันทึก ข้อมูลจะวิ่งเข้าเงื่อนไข `else` เสมอ ซึ่งจะทำการ `push` ลูกค้าคนเดิมซ้ำเข้าไปเรื่อยๆ ทำให้ข้อมูลใน `localStorage` มีลูกค้าชื่อเดียวกันซ้ำซ้อนหลายแถว
    *หมายเหตุ: แม้ว่าในขั้นตอนการแสดงผลบน UI ตัวแอปจะมีการขจัดตัวซ้ำด้วยชื่อผ่าน `renderQuickClientsDropdown` อยู่แล้ว แต่ในตัวแปรดิบที่บันทึกอยู่ใน LocalStorage จะบวมขึ้นทุกครั้งที่กดบันทึก*
  - **แนวทางแก้ไข (เสนอแนะเพื่อปรับปรุงในอนาคต):**
    หากต้องการป้องกันข้อมูลขยะสะสมในอนาคต สามารถปรับลอจิกการหาตัวซ้ำให้ครอบคลุมชื่อลูกค้าในกรณีไม่มี Tax ID ดังนี้ค่ะ:
    ```javascript
    const existingIdx = customClients.findIndex(c => 
      (taxId !== '' && c.taxId === taxId) || (taxId === '' && c.name === name)
    );
    const existsInDefaults = DEFAULT_CLIENTS.some(c => 
      (taxId !== '' && c.taxId === taxId) || (taxId === '' && c.name === name)
    );
    ```
