- **1. การบันทึกและซิงค์ภาษีหัก ณ ที่จ่าย (Withholding Tax 3% Auto-Recording - PASS):**
  - **การแกะรอยโค้ด (Path Tracing):** ในฟังก์ชัน Event Listener สำหรับแบบฟอร์มเอกสารหน้า Slide 4 ([super_app_anime.html:L6345-6355](file:///Users/chz/Desktop/ChZ_Agent_Corp/Agent_Lab/super_app_anime.html#L6345-6355)) พี่คิวได้ปรับโครงสร้างการประกาศออบเจกต์ธุรกรรม `newTx` เป็นดังนี้:
    ```javascript
    const newTx = {
      id: Date.now(),
      date: date,
      time: new Date().toTimeString().split(' ')[0].substring(0, 5),
      type: 'income',
      category: 'Feltz Studio',
      remarks: `[${type === 'invoice' ? 'ใบวางบิล' : 'ใบเสร็จรับเงิน'}] ${docNumber} - ${client}${hasTax3 ? ' (หัก ณ ที่จ่าย 3%)' : ''}`,
      amount: netTotal,
      hasTax: hasTax3,
      syncStatus: 'local'
    };
    ```
  - **การพิสูจน์ผลลัพธ์ (Verification):**
    1. ออบเจกต์ `newTx` มีการกำหนดแอตทริบิวต์ `hasTax` โดยสอดรับกับตัวแปรสถานะเช็กบ็อกซ์ `hasTax3` อย่างถูกต้อง
    2. ฟิลด์ `remarks` มีการตรวจสอบเงื่อนไข หากมีการหักภาษีจริงจะพ่นคำอธิบายกำกับ `(หัก ณ ที่จ่าย 3%)` เพิ่มเข้าไปท้ายข้อความแบบไดนามิกโดยอัตโนมัติ
    3. เมื่อนำไปตรวจสอบร่วมกับปลายน้ำของลอจิกการส่งข้อมูลขึ้นคลาวด์ [sendTransactionToGAS](file:///Users/chz/Desktop/ChZ_Agent_Corp/Agent_Lab/super_app_anime.html#L5209-5250) ตัวแปร `payload.hasTaxWithholding` จะถูกแมปข้อมูลจาก `tx.hasTax` และคำค้นหาคำอธิบายใน `tx.remarks` เป็นหลัก ส่งผลให้เมื่อธุรกรรมนี้ถูกซิงค์ออนไลน์ ข้อมูลสถานะการเสียภาษีหัก ณ ที่จ่าย 3% บนระบบ Google Sheets บัญชีกลางของบริษัทจะถูกต้อง 100% แล้วค่ะ
